# CONTRIBUTING

*Like this project is young, there is no (yet) precise rules.*

Just follow **usual best practices**:
* *Explain* fix or feature into *pull request*
* Implementation:
    * branch from `master`
    * produce *beautiful* source code
    * *format* source code
* Update:
    * *tests*
    * *documentation*
    * *sample*
* Commit:
    * *squash*
    * format *message*
* ...
